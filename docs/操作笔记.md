## 四、全市场扫描

### 离线批量扫描（推荐，约 30 分钟）

```python
# run_scan.py
import asyncio
from data.downloader import DataDownloader

async def main():
    downloader = DataDownloader()
    results = await downloader.screen_all(market="CN")

asyncio.run(main())
```

运行：
```powershell
python run_scan.py
```

扫描结果输出示例：
```
Scan complete: 5201 stocks
Signal breakdown:
  no_entry: 4672
  extended: 479
  buy: 46
  watch: 4
```

### 在线 API 扫描（不推荐，会超时）

`POST /api/screen/run` — 扫描 5200 只股票需要 10 分钟以上，HTTP 连接会超时。

### 扫描缓存说明

- 离线扫描结果保存在进程内存 `web.scan_cache._scan_cache` 中
- **后端重启后缓存丢失**，需要重新扫描
- 建议方案：将结果持久化到文件或数据库的 `screen_results` 表

## 五、买入信号解读

### 信号分类

| 信号 | 含义 | 操作建议 |
|------|------|---------|
| `buy` | 已识别形态并站上买入点 | 确认成交量后可入场 |
| `watch` | 已识别形态，但未站上买入点 | 等待放量突破 |
| `extended` | 形态已延伸（超出买入点过远） | 等待回调/新形态 |
| `no_entry` | 不满足条件 | — |

### 评分体系

| 分数 | 含义 |
|------|------|
| 55 | 高强度信号：RS≥80 + Stage2 + 形态识别 + 成交量确认 |
| 52 | 标准信号：基础条件满足但部分指标偏弱 |

### 形态类型

目前仅支持 **双底（Double Bottom）** 形态识别：
- 第一底 / 第二底自动识别
- 颈线（买入点）= 两底之间的最高点
- 止损 = 第二底下方 ATR 倍数
- 目标 = 买入点 + 形态高度

## 八、常见问题

### Q: 前端显示 price=0 / 数据为空？
A: 离线扫描缓存丢失。重新运行离线扫描，或直接在 API 查询个股（`/api/stock/{code}`）。

### Q: 批量扫描太慢？
A: 5200 只股票约 30 分钟。这是正常的——每只股票需要计算 RS、Stage2、形态识别等多个步骤。使用离线脚本避免 HTTP 超时。

### Q: akshare 连不上？
A: 东方财富/Sina API 在中国境外可能被限。可以重试或使用代理。国内网络环境下通常稳定。

### Q: 股票名称乱码？
A: PowerShell 控制台编码问题。使用 `sys.stdout.reconfigure(encoding='utf-8')` 或通过 JavaScript 前端查看（前端名称显示正常）。

### Q: 如何添加自选股？
A: 通过前端看板的搜索功能找到股票，点击加入自选股。或直接在数据库 `watchlist` 表插入。

## 九、项目配置

### 评分参数（`config/`）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| RS 阈值 | 80 | 相对强度最低分 |
| Stage2 确认 | 是 | 是否处于第二阶段 |
| 最小成交量比 | 1.5 | 突破日最低量比 |
| 延伸上限 | 5% | 超出买入点多少就算 extended |
| 形态最小高度 | 15% | 双底形态最小波动幅度 |
